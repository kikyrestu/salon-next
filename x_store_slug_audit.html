⚠️ 90% halaman frontend mengirim API call tanpa x-store-slug — semua akan fallback ke database "pusat"
Bagaimana bug ini terjadi
1
User login sebagai toko salon-abc
Session menyimpan tenantSlug = "salon-abc"
2
Halaman memanggil fetch("/api/customers")
Tanpa header x-store-slug — bug!
3
API route melakukan fallback
request.headers.get('x-store-slug') || 'pusat'
4
Data yang tampil = data toko "pusat"
Bukan data salon-abc — data isolation rusak!
Risiko berdasarkan tingkat keparahan
🔴 Kritis — Kebocoran data antar tenant
Toko selain "pusat" akan membaca & menulis ke database toko lain. Customer, invoice, transaksi bercampur.
🔴 Kritis — Korupsi data saat write/edit/delete
Simpan customer baru, hapus produk, update settings → masuk ke database toko "pusat", bukan toko yang login.
🟠 Tinggi — Multi-tenancy tidak berfungsi
Seluruh arsitektur multi-tenant jadi sia-sia. Semua toko non-pusat melihat data yang sama.
🟠 Tinggi — RBAC bypass tidak terdeteksi
lib/rbac.ts memvalidasi x-store-slug vs session — tanpa header, validasi ini dilewati sepenuhnya.
🔵 Medium — Hanya "pusat" yang kerja normal
Secara tidak sengaja, toko "pusat" adalah satu-satunya toko yang berfungsi benar saat ini.

Rekomendasi perbaikan
Pola yang sudah benar (contoh dari appointments/page.tsx):
const headers = { "x-store-slug": slug }; fetch("/api/customers", { headers }) fetch("/api/services", { headers }) fetch("/api/products", { headers })
Slug sudah tersedia via Next.js params di setiap halaman:
const { slug } = await params; // atau const params = useParams(); const slug = params.slug as string;
